# Seg: Микросервис сегментации пользователей

REST API сервис для управления пользователями и их сегментацией. Позволяет назначать сегменты (тэги/группы) пользователям, а также проводить массовое распределение (например, для A/B тестирования) на заданный процент аудитории.

## Архитектурные решения

- Взаимодействие с базой данных реализовано через чистый SQL. Написана собственная легковесная система миграций.
- Интегрирован Redis для кэширования запросов и правильной инвалидации данных.
- Жесткий контроль стиля, линтинг и статическая типизация с помощью Ruff, wemake-python-styleguide и Mypy.

## Технологи

- Бэкенд: Python 3.13, `uv`, FastAPI
- БД: PostgreSQL, Redis
- Инфра: Docker, Docker Compose

## Запуск проекта

Приложение требует запущенной базы данных, предоставлены конфигурации для поднятия необходимых сервисов локально, и полностью контейнеризованного запуска.

Для обоих потребуется конфигурационный файл `.env` в корне проекта (пример для локальной среды):
```ini
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres
```

### Запуск через Docker

```bash
docker compose --file compose-prod.yml up --build
```

### Запуск для локальной разработки

```bash
docker compose up --build
```

Установите зависимости проекта, примените миграции и запустите dev-сервер FastAPI:
```bash
uv sync
uv run seg/main.py --verbose
uv run seg/app.py
```

Интерактивная документация OpenAPI (Swagger) будет доступна по адресу `http://localhost:8000/docs`.

## Примеры использования

1. Создание новых сегментов

```bash
curl -X 'POST' 'http://localhost:8000/api/v1/s/' \
  -H 'Content-Type: application/json' \
  -d '[
  {"name": "CLOUD_DISCOUNT_30"},
  {"name": "MAIL_GPT"}
]'
```

2. Массовое назначение сегмента

Автоматическое распределение сегмента на случайные 50% от текущей базы пользователей:

```bash
curl -X 'POST' 'http://localhost:8000/api/v1/su/mass' \
  -H 'Content-Type: application/json' \
  -d '{
  "ratio": 0.5,
  "segment_ids": ["c39848ba-05b2-42e0-a63f-541cbc0e3646"]
}'
```

3. Получение списка пользователей в сегменте

```bash
curl -X 'GET' 'http://localhost:8000/api/v1/u/?segment_id=c39848ba-05b2-42e0-a63f-541cbc0e3646'
```
